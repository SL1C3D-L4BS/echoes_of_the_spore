# Developer Workflow

## Daily Development Loop

```
1. Pull latest from develop
2. Create feature branch: feature/ECS-movement-system
3. Read relevant bible section(s)
4. Write code + tests in parallel
5. Run EditMode tests locally
6. Commit with conventional commit message
7. Push, create PR to develop
8. CI runs automatically
9. Code review (or Claude Code review)
10. Merge to develop
```

## Conventional Commits

```
feat: add MovementSystem with gravity and terminal velocity
fix: correct FixedPoint64 overflow in multiplication
docs: update combat damage formula in bible
test: add determinism test for 1000-tick simulation
refactor: extract SpatialHashGrid from CollisionSystem
perf: optimize snapshot serialization with Span<byte>
chore: update Unity to 6.3.1f1
```

## Claude Code Workflow

```bash
# Navigate to project
cd C:\DEV\game-dev\unity\projects\echoes_of_the_spore

# Launch Claude Code (auto-loads CLAUDE.md)
claude

# Claude reads bible, writes code, runs tests
# Example: "Build the MovementSystem per 13-tech-architecture"
```

Claude Code will:
1. Read the relevant bible section(s) from the routing table
2. Write production-ready C# code
3. Write corresponding tests
4. Verify no floats in Core/
5. State what to build next per dependency map

## IDE Setup

### VS Code (Primary)
- Open project folder (not just eots/)
- Extensions: C# Dev Kit, Claude Code, Unity Tools
- Terminal: Git Bash default
- Settings: Space Mono font, SL1C3D-L4BS theme

### Visual Studio 2022 (C# + Debugging)
- Open eots.sln (generated by Unity)
- Attach to Unity Editor for breakpoint debugging
- Workloads: C# .NET Desktop, Unity Game Dev
