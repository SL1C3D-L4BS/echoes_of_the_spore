# Tilemap Instantiation

## Overview

Tilemap instantiation converts `ZoneGraphData` (generated by Rust) into playable Unity rooms. This process runs on the Unity main thread and must complete within the loading screen window. Each room is instantiated as a set of GameObjects with Tilemap components, enemy spawners, hazards, and interactables.

## Pipeline

```
ZoneGraphData (from ProcGenBridge)
  |
  +-- RoomInstantiator.InstantiateZone()
        |
        +-- For each RoomData in graph:
        |     |
        |     +-- 1. Prefab Lookup (zone_id + room_type -> prefab)
        |     +-- 2. Tilemap Population (collision grid -> Unity Tilemap)
        |     +-- 3. Enemy Spawning (spawner list -> SpawnerComponent entities)
        |     +-- 4. Hazard Placement (hazard list -> HazardComponent entities)
        |     +-- 5. Door Connection (edge list -> DoorComponent entities)
        |     +-- 6. Decoration Pass (visual props, parallax, lighting)
        |
        +-- Room Activation System (enable/disable based on player position)
```

## Prefab Lookup

### Prefab Database

| Key | Value | Location |
|-----|-------|----------|
| (zone_id, room_type) | List of prefab variants | `RoomPrefabDatabase` ScriptableObject |
| Selection | Room seed selects variant from list | Deterministic per room |

### Prefab Structure

Each room prefab contains:

```
RoomPrefab (GameObject)
  +-- Tilemap_Ground (Tilemap component, collision layer)
  +-- Tilemap_Platforms (Tilemap component, one-way collision)
  +-- Tilemap_Decoration (Tilemap component, no collision)
  +-- SpawnerParent (empty, spawner children placed procedurally)
  +-- HazardParent (empty, hazard children placed procedurally)
  +-- DoorParent (empty, door children at edge positions)
  +-- LightParent (URP 2D lights, zone-specific preset)
  +-- ParallaxLayers (5 SpriteRenderer children, zone background art)
  +-- RoomBounds (BoxCollider2D trigger for room activation)
```

### Prefab Variant Counts

| Room Type | Variants Per Zone | Total Prefabs (8 zones) |
|-----------|------------------|------------------------|
| Combat | 5-8 | 48 |
| Puzzle | 2-3 | 20 |
| Reward | 2 | 16 |
| Rest | 2 | 16 |
| Boss | 1 (hand-authored) | 8 |
| Transition | 1 | 8 |
| Secret | 3 | 24 |
| **Total** | -- | **~140** |

## Tilemap Population

### From Collision Grid to Unity Tilemap

1. Receive `collision_grid: Vec<u8>` from Rust (bit-packed: 0 = air, 1 = solid, 2 = platform).
2. Iterate grid row by row.
3. For each cell:
   - 0 (air): skip.
   - 1 (solid): place ground/wall Rule Tile on `Tilemap_Ground`.
   - 2 (platform): place platform Rule Tile on `Tilemap_Platforms`.
4. Rule Tiles auto-connect edges based on neighbor state.
5. Decoration pass: place detail tiles on `Tilemap_Decoration` using room seed.

### Tile Asset Lookup

| Tile Type | Asset | Auto-Tile |
|-----------|-------|-----------|
| Ground | `ZoneGroundRuleTile` (per zone) | 47-tile blob set |
| Wall | `ZoneWallRuleTile` (per zone) | 47-tile blob set |
| Platform | `ZonePlatformRuleTile` (per zone) | 15-tile subset |
| Decoration | `ZoneDecoTile` (per zone) | Random from set |

### Performance

| Metric | Target |
|--------|--------|
| Tiles per room (avg) | 2,000 - 4,000 |
| Tilemap set time per room | < 50ms |
| Total zone tilemap time | < 500ms (10-25 rooms) |

## Enemy Spawning

### From SpawnerList to ECS Entities

1. Receive `spawners: Vec<SpawnerData>` for each room.
2. For each spawner:
   - Create `SpawnerEntity` with components: `Position`, `EnemyType`, `ActivationTrigger`, `FacingDirection`.
   - Parent to room's SpawnerParent GameObject.
   - Enemy prefab reference resolved from `EnemyDatabase` by (zone_id, enemy_type, tier).
3. Spawner activation handled by `SpawnerActivationSystem` (see ROOM_GENERATION.md).

### Enemy Prefab Resolution

| Input | Lookup |
|-------|--------|
| zone_id + enemy_type | `EnemyDatabase[zone_id][enemy_type]` returns prefab reference |
| tier | Tier determined by room difficulty value |
| variant | Room seed selects visual variant (recolor, size modifier) |

## Hazard Placement

### From HazardList to GameObjects

1. Receive `hazards: Vec<HazardData>` for each room.
2. For each hazard:
   - Instantiate hazard prefab from `HazardDatabase[zone_id][hazard_type]`.
   - Set position, size, timing parameters from HazardData.
   - Parent to room's HazardParent.
3. Hazard behavior driven by `HazardSystem` (damage on overlap, timing patterns).

### Hazard Prefab Components

| Component | Purpose |
|-----------|---------|
| `HazardComponent` | Type, damage, tick rate |
| `BoxCollider2D` (trigger) | Damage zone detection |
| `SpriteRenderer` | Visual representation |
| `VFXController` | Particle effects (acid bubbles, fire, etc.) |
| `HazardTimingComponent` | On/off cycle timing (if applicable) |

## Room Activation / Deactivation

### Culling Strategy

Only the current room and its immediate neighbors are active. All other rooms are deactivated.

| Room State | Active | Rendered | Physics | Enemies |
|-----------|--------|----------|---------|---------|
| Current | Yes | Yes | Yes | Active |
| Adjacent (1 edge away) | Yes | Yes | No | Frozen |
| Distant (2+ edges) | No | No | No | Despawned |

### Activation Trigger

```
Player enters RoomBounds trigger -> RoomActivationSystem
  -> Activate current room (full)
  -> Activate adjacent rooms (render only)
  -> Deactivate all others
  -> Fire RoomEntered event (consumed by minimap, narration, etc.)
```

### Memory Management

| Metric | Target |
|--------|--------|
| Active room memory | < 20MB (tiles + entities + VFX) |
| Adjacent room memory | < 10MB each (tiles only, no entities) |
| Total zone memory | < 80MB (all rooms loaded, most deactivated) |
| Activation time | < 16ms (within single frame budget at 60fps) |

## Door Connection

### From EdgeData to Door GameObjects

1. For each edge in ZoneGraphData:
   - Determine wall position in source room (left/right/top/bottom).
   - Instantiate Door prefab at wall position.
   - Set DoorComponent: target_room_id, edge_type, locked state.
2. Door pairs (source room door + destination room door) are cross-referenced.
3. Transition: player enters door trigger -> fade to black (0.3s) -> activate target room -> place player at target door -> fade in (0.3s).

### Door Prefab Components

| Component | Purpose |
|-----------|---------|
| `DoorComponent` | Target room, edge type, lock state |
| `BoxCollider2D` (trigger) | Player entry detection |
| `SpriteRenderer` | Door visual (open/locked/secret) |
| `DoorBarrierVFX` | Energy barrier effect when locked |
| `AudioSource` | Door open/close SFX |

## Decoration Pass

After structural elements are placed, a decoration pass adds visual variety:

| Element | Placement Rule | Count Per Room |
|---------|---------------|---------------|
| Floor props | Random on ground tiles (vines, cracks, debris) | 5-15 |
| Wall props | Random on wall tiles (moss, pipes, growths) | 3-10 |
| Ceiling props | Random on ceiling tiles (stalactites, roots) | 2-8 |
| Hero piece | Centered in room if room type permits (see ENVIRONMENT_ART.md) | 0-1 |
| Light props | Near hero pieces and doors | 2-6 |

All decoration placement uses the room's visual sub-seed (salt 0x05) for determinism.

## Implementation Notes

- `RoomInstantiator` in `Assets/_Project/Scripts/ProcGen/RoomInstantiator.cs`
- Prefab database: `Assets/_Project/Data/RoomPrefabDatabase.asset`
- Room activation: `RoomActivationSystem` (ECS) checks player position against room bounds each tick
- Door transitions managed by `DoorTransitionController` MonoBehaviour with async scene-like fade
- Decoration props placed by `DecorationPlacer` using Burst-compiled tile iteration
- All instantiation runs during loading screen, progress reported to loading bar via callback
- Addressable assets: room prefabs loaded per-zone using Addressable labels (`zone_1_rooms`, etc.)
