## Soft Gates vs Hard Gates

| Gate Type | Examples | Design Intent |
|-----------|----------|---------------|
| **Hard gate** | Zone requires Community Tree depth | Pacing control, ensures server has players |
| **Soft gate** | Boss difficulty, Heat system | Skill-based, player chooses when ready |
| **Social gate** | Fusion (requires partner), Rite (requires team) | Encourages multiplayer engagement |
| **Knowledge gate** | Codex entries (require experimentation) | Rewards curiosity and exploration |

## Anti-Frustration Rules

1. **No gear checks** — any class/Crest can clear any zone at Heat 0
2. **No mandatory grinding** — skill can overcome stat deficiency
3. **No time gates** — Community Tree progresses by contribution, not real-time waiting
4. **No daily login requirements** — returning players get catch-up Echo multiplier
5. **Solo is always viable** — Micro-Plan replaces Spore Plan, AI fills co-op gaps with increased boon quality
