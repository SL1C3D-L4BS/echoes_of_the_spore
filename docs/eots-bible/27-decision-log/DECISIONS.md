# 27 — DECISION LOG

| ID | Decision | Alternatives Considered | Rationale | Date | Status |
|---|---|---|---|---|---|
| D01 | Use Unreal Engine 5.5 for rendering, custom ECS for simulation | Unity, Godot, fully custom engine | UE5 provides Nanite for parallax depth, mature console export pipeline, and industry-standard tooling. Custom ECS required because UE5 gameplay framework is not deterministic. | Apr 2026 | Accepted |
| D02 | FixedPoint64 (32.32 int64) for all simulation math | IEEE 754 float32, double, fixed 16.16 | Float32 is non-deterministic across platforms. Double is overkill and slower. 16.16 lacks range for world coordinates. 32.32 provides sufficient range (±2 billion integer, 1/4-billion fractional precision) with guaranteed cross-platform determinism. | Apr 2026 | Accepted |
| D03 | Rust for procedural generation | C++, Python, JavaScript | Rust provides: memory safety without GC (critical for game tooling), no_std support for WASM target, excellent performance matching C++, strong type system for constraint validation. C++ is viable but Rust's safety guarantees reduce proc-gen bug risk. | Apr 2026 | Accepted |
| D04 | Supabase + Redis (not Firebase, not custom SQL) | Firebase, raw PostgreSQL, DynamoDB | Supabase provides: PostgreSQL with PL/pgSQL for atomic operations, Realtime subscriptions for Community Tree sync, Row-Level Security. Redis provides: sub-millisecond session operations, Lua scripting for atomic state machines. Firebase lacks PL/pgSQL power. Raw PostgreSQL lacks Realtime. DynamoDB is overkill and expensive. | Apr 2026 | Accepted |
| D05 | 60Hz fixed tick rate (not 30Hz, not 120Hz) | 30Hz, 120Hz, variable | 30Hz feels laggy for action combat. 120Hz doubles simulation cost with minimal perceived benefit in 2D. 60Hz is the standard for action games and aligns with common display refresh rates. | Apr 2026 | Accepted |
| D06 | Rollback netcode (not lockstep, not server-authoritative-only) | Lockstep, pure server-auth | Lockstep requires all players to have identical latency (impractical). Pure server-auth adds input latency equal to round-trip (unacceptable for action combat). Rollback provides instant local response with correction on mismatch. | Apr 2026 | Accepted |
| D07 | 4 classes at launch (not 6, not 8) | 6 classes, 8 classes | Each class requires: 3 Crests, 8 Functions (each with 3 modes), sprite sheets with 5 infection tiers, animations. 4 classes already produce 12 Fused Classes. 6 would produce 30 (unmanageable). 4 is the sweet spot for depth vs scope. | Apr 2026 | Accepted |
| D08 | Ship 8 zones (with 4-zone minimum viable product) | 4 zones, 6 zones, 12 zones | 4 zones is the minimum for a satisfying Metroidvania loop (start → 2 mid-game paths → final). 8 zones is the aspirational target. 12 is out of scope for $10M budget. | Apr 2026 | Accepted |
| D09 | Premium $29.99 base price (not F2P, not $19.99, not $39.99) | F2P, $19.99, $39.99 | F2P requires aggressive monetization that conflicts with anti-predatory principles. $19.99 undervalues the product given content depth. $39.99 is above indie price ceiling for new IP. $29.99 matches Mewgenics and Hades II successfully. | Apr 2026 | Accepted |
| D10 | No native mobile build (cloud-streaming only) | Native mobile port, no mobile at all | Native mobile would require significant control redesign and performance optimization. Cloud-streaming via Xbox Cloud Gaming / GeForce NOW provides mobile access without additional development cost. | Apr 2026 | Accepted |
